# 注意点・改善候補

対象アプリ: `zend-framework-1-crud-master`

このページでは、引き継ぎ時に先に伝えるべき注意点と、今後直すなら優先したい改善候補をまとめる。

## 前提

- このアプリはZend Framework 1系のCRUDサンプル。
- ZF1は古いフレームワークで、現在の新規開発向きではない。
- まずは学習・解析・引き継ぎ資料として扱い、実運用に載せる場合は改修が必要。

## 改善候補

### 1. 削除処理をPOST化する

現在は `/ticket/delete/:id` にGETでアクセスすると削除される。

改善案:

- 一覧画面の削除リンクを確認画面またはPOSTフォームに変える。
- `deleteAction()` でGET削除を禁止する。
- CSRF対策を入れる。

関連:

- [05_URL一覧.md](05_URL一覧.md)
- [ticket/index.phtml 解説](各ファイル解説/application/views/scripts/ticket/index.phtml_解説.md)
- [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md)

### 2. DB接続情報を環境ごとに分離する

`application.ini` にDBユーザー名、パスワード、DB名が書かれている。

改善案:

- 本番・開発・テストで設定を分ける。
- パスワードをリポジトリに含めない。
- 環境変数や外部設定ファイルを使う。

関連:

- [application.ini 解説](各ファイル解説/application/configs/application.ini_解説.md)

### 3. アップロード処理を見直す

登録/編集フォームにファイルアップロードがある。

改善案:

- 保存先ディレクトリの公開範囲を確認する。
- MIMEタイプ検証を追加する。
- ファイル名生成と保存後の扱いを整理する。
- 編集時にファイル必須でよいか見直す。

関連:

- [TopicBootstrapForm.php 解説](各ファイル解説/application/forms/TopicBootstrapForm.php_解説.md)
- [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md)

### 4. 未使用ファイルを整理する

このプロジェクトには、現在のチケットCRUDでは使われていない可能性が高いファイルがある。

候補:

- `application/forms/----TopicForm.php`
- `application/views/helpers/DisplayAddress.php`
- `application/views/helpers/DisplayDateLab.php`
- `application/views/helpers/LoggedInUser.php`
- `application/views/helpers/menuList.php`
- `application/helpers/Multiples.php`

関連:

- [07_共通クラス・関数.md](07_共通クラス・関数.md)

### 5. 表記ゆれを直す

画面上の `CVS export` は、内容から見ると `CSV export` の意味だと思われる。

改善案:

- ナビゲーション表示を `CSV export` に変更する。
- Action名 `cvsexportAction()` は影響範囲を見て変更するか判断する。

関連:

- [layout.phtml 解説](各ファイル解説/application/layouts/scripts/layout.phtml_解説.md)
- [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md)

## 引き継ぎ時に先に伝えること

- これはZF1の古いサンプルアプリである。
- 主要機能はチケットの一覧、登録、編集、削除、CSV出力。
- 実行入口は `public/index.php`。
- URL定義は `application/configs/routes.php`。
- 中心Controllerは `TicketController.php`。
- DB操作の中心は `TicketMapper.php`。
- 画面は `application/views/scripts/ticket/` 配下。
- 実運用化するなら、削除、DB設定、アップロード、依存ライブラリ確認を優先して見る。
