# 脆弱性簡易診断

対象アプリ: `zend-framework-1-crud-master`

このページは、ソースコードを読んだ範囲での簡易診断メモ。実際の脆弱性診断ツールや動作環境での検査は行っていない。

## 診断サマリ

| 重要度 | 項目 | 状態 |
| --- | --- | --- |
| 高 | ZF1の利用 | 古いフレームワークのため要注意 |
| 高 | GETによる削除 | `/ticket/delete/:id` でGET削除される |
| 高 | DB接続情報 | `application.ini` に直書き |
| 中 | ファイルアップロード | 拡張子制限はあるが追加確認が必要 |
| 中 | SQL条件 | 一部で文字列連結の条件がある |
| 中 | XSS対策 | 一部escapeあり。一部は要確認 |
| 低 | 未使用コード | 認証系Helperなど未接続コードが残る |

## 高リスク

### 1. Zend Framework 1の利用

ZF1は古いフレームワークで、現在の保守・セキュリティ更新を前提にしづらい。

影響:

- 新しいPHP環境で動作しない可能性がある。
- 既知脆弱性や依存ライブラリの問題を別途確認する必要がある。
- 新規開発ではなく、学習・移行・保守対象として扱うべき。

関連:

- [01_ZendFramework1について.md](01_ZendFramework1について.md)
- [library.rar 解説](各ファイル解説/library.rar_解説.md)

### 2. GETで削除できる

`/ticket/delete/:id` にGETアクセスすると削除処理が実行される。

影響:

- URLを踏むだけで削除される。
- 外部ページやメール内リンクから意図せず削除される可能性がある。
- CSRFの影響を受けやすい。

改善案:

- 削除はPOSTだけにする。
- 削除確認画面を挟む。
- CSRFトークンを検証する。

関連:

- [05_URL一覧.md](05_URL一覧.md)
- [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md)
- [ticket/index.phtml 解説](各ファイル解説/application/views/scripts/ticket/index.phtml_解説.md)

### 3. DB接続情報の直書き

`application/configs/application.ini` にDB接続情報が書かれている。

影響:

- リポジトリ共有時に接続情報が漏れる可能性がある。
- 本番・開発・テストの設定差分を管理しづらい。

改善案:

- パスワードをリポジトリに含めない。
- 環境変数や外部設定に分離する。
- 本番環境では最小権限のDBユーザーを使う。

関連:

- [application.ini 解説](各ファイル解説/application/configs/application.ini_解説.md)

## 中リスク

### 4. ファイルアップロード処理

`TopicBootstrapForm` で `jpg,png,gif` の拡張子制限がある。

確認したい点:

- MIMEタイプや実ファイル内容の検証が十分か。
- 保存先がWeb公開領域から直接実行されないか。
- ファイル名衝突や上書きが起きないか。
- 編集時もファイル必須でよいか。

関連:

- [TopicBootstrapForm.php 解説](各ファイル解説/application/forms/TopicBootstrapForm.php_解説.md)
- [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md)
- [application/uploads](各ファイル解説/application/uploads/avatar92.jpg_解説.md)

### 5. SQL条件の文字列連結

`TicketMapper#deleteTopic($id)` で削除条件が文字列連結になっている。

```php
return $this->getDbTable()->delete("id = $id");
```

このアプリではルート側で `id` を数字に制限しているが、Mapper単体としては安全な書き方ではない。

改善案:

- `array('id = ?' => $id)` のようなプレースホルダ形式にする。
- Mapperの入口でもIDを整数化する。

関連:

- [TicketMapper.php 解説](各ファイル解説/application/models/TicketMapper.php_解説.md)
- [routes.php 解説](各ファイル解説/application/configs/routes.php_解説.md)

### 6. XSS対策の確認

一覧画面では `notes` はescapeされているが、`title` はそのまま出力されている。

確認したい点:

- `title`、FlashMessage、エラー表示など、ユーザー入力由来の値がescapeされているか。
- Viewテンプレート全体で出力時のルールが統一されているか。

関連:

- [ticket/index.phtml 解説](各ファイル解説/application/views/scripts/ticket/index.phtml_解説.md)
- [layout.phtml 解説](各ファイル解説/application/layouts/scripts/layout.phtml_解説.md)

## 低リスク・整理対象

### 7. 未使用コードや流用元コード

認証表示Helper、住所表示Helper、旧フォームなど、現在のCRUD本流では使われていない可能性が高いコードがある。

影響:

- 引き継ぎ時に読み手が迷いやすい。
- 将来の改修時に誤って使う可能性がある。

関連:

- [07_共通クラス・関数.md](07_共通クラス・関数.md)
- [09_注意点・改善候補.md](09_注意点・改善候補.md)

## 優先して直すなら

1. 削除処理をPOST化する。
2. DB接続情報を環境変数または外部設定へ分離する。
3. アップロード処理の検証を強化する。
4. 一覧画面の出力をすべてescapeする。
5. 未使用ファイルを整理する。

## 診断の限界

このページは静的なソース確認に基づく簡易診断。

未実施:

- 実行環境での動作確認
- 依存ライブラリのCVE照合
- 認証・権限の実テスト
- アップロードファイルの実検証
- SQLインジェクションやXSSの実攻撃テスト

本格的に運用判断する場合は、実行環境を用意して動的診断と依存関係チェックを行う。
