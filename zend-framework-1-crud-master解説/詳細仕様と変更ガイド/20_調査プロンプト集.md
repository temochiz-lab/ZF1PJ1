# 調査プロンプト集

対象アプリ: `zend-framework-1-crud-master`

## 目的

このページは、VaultとソースをCodexで再調査するときのプロンプト例をまとめる。

## 変更影響調査

```text
このVaultとソース全体を見て、
「○○を変更する場合」
修正が必要なファイル、理由、修正内容、注意点を列挙して。
```

```text
このVaultとソース全体を見て、
「ticketsテーブルに is_done カラムを追加する場合」
DB / Model / Mapper / Form / Controller / View / CSV / Docs に分けて、
修正対象を列挙して。
```

## 共通クラス変更

```text
このVaultとソース全体を見て、
「TicketMapper.php を変更する場合」
影響する画面、Controller、DB処理、CSV出力、ドキュメントを列挙して。
```

```text
このVaultとソース全体を見て、
「DisplayDate.php の表示形式を変更する場合」
影響するViewと確認すべき画面を列挙して。
```

## エラー・ログ調査

```text
このVaultとソース全体を見て、
現在のエラー表示とログ出力の仕組みを整理して。
実際にログファイルへ出力されるかも確認して。
```

```text
このVaultとソース全体を見て、
ファイルアップロード失敗時にどのようなエラー表示・ログ出力になるか調査して。
```

## セキュリティ調査

```text
このVaultとソース全体を見て、
削除処理をGETからPOSTに変更する場合の修正箇所と注意点を列挙して。
```

```text
このVaultとソース全体を見て、
XSSの可能性がある表示箇所を列挙して。
エスケープ済み/未エスケープも分けて。
```

## ドキュメント更新

```text
今の変更内容に合わせて、
関連する解説ページ、各ファイル解説、URL一覧、DB資料を更新して。
対象外のファイルは触らないで。
```

```text
この改修内容を引き継ぎ資料として残したい。
変更理由、修正箇所、影響範囲、今後の注意点をmdに追記して。
```

## GitHub保存

```text
今回の変更を確認して、コミットしてGitHubにpushして。
コミットメッセージは「○○」にして。
```

## 使い方の注意

- 「全体を見て」と明示する。
- 分類して列挙させる。
- 変更理由、修正箇所、注意点をセットで聞く。
- 実装まで任せる場合は「対象外のファイルは触らないで」と明示する。
- 変更後はドキュメント更新も依頼する。
