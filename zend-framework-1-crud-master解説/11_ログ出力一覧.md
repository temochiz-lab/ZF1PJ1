# ログ出力一覧

対象アプリ: `zend-framework-1-crud-master`

## 結論

このアプリには、通常処理の業務ログやアクセスログを明示的に出力する実装はほぼない。

ログ出力として確認できるのは、主に `ErrorController` からZend Frameworkの `Log` リソースへ出す例外ログ。ただし、[`application.ini`](各ファイル解説/application/configs/application.ini_解説.md) には `resources.log` の設定が見当たらないため、実際にログファイルへ出力されるとは限らない。

## ログ出力箇所

| ファイル | 箇所 | 内容 | 備考 |
| --- | --- | --- | --- |
| [`application/controllers/ErrorController.php`](各ファイル解説/application/controllers/ErrorController.php_解説.md) | `errorAction()` | エラーメッセージをログ出力 | `getLog()` がログリソースを取得できた場合のみ |
| [`application/controllers/ErrorController.php`](各ファイル解説/application/controllers/ErrorController.php_解説.md) | `errorAction()` | リクエストパラメータをログ出力 | 例外発生時の調査用 |

## ErrorControllerのログ処理

```php
if ($log = $this->getLog()) {
    $log->log($this->view->message, $priority, $errors->exception);
    $log->log('Request Parameters', $priority, $errors->request->getParams());
}
```

処理の意味:

- 404系エラーでは `Zend_Log::NOTICE` を使う。
- アプリケーションエラーでは `Zend_Log::CRIT` を使う。
- 例外情報とリクエストパラメータをログへ渡す。
- ただし `getLog()` が `false` を返す場合はログ出力されない。

## ログリソースの確認

`getLog()` はBootstrapから `Log` リソースを取得しようとする。

```php
$bootstrap = $this->getInvokeArg('bootstrap');
if (!$bootstrap->hasResource('Log')) {
    return false;
}
$log = $bootstrap->getResource('Log');
```

しかし、[`application/configs/application.ini`](各ファイル解説/application/configs/application.ini_解説.md) には `resources.log` や `resources.logger` のような設定が見当たらない。

そのため、このままでは `hasResource('Log')` が `false` になり、ログは出力されない可能性が高い。

| チケット登録成功 | success | `Ticket has been successfully posted.` |
| チケット登録失敗 | danger | `Some error occurred while posting tickets, Please try after some time.` |

## 改善候補

- [`application.ini`](各ファイル解説/application/configs/application.ini_解説.md) にログ出力先を設定する。
- ファイルログを出す場合は、`data/logs/application.log` のような保存先を決める。
- 登録・更新・削除・CSV出力にも操作ログを追加する。
- エラー時にユーザー向けメッセージと管理者向けログを分ける。
- ファイルアップロード失敗時の例外を握りつぶさずログ出力する。
