## 主要な配置

| パス | 役割 |
| --- | --- |
| `public/index.php` | Webアプリの入口。全リクエストをZend Frameworkへ渡す。 |
| `public/css/app.global.css` | 画面用CSS。Bootstrap系スタイルと独自CSSを含む。 |
| `application/Bootstrap.php` | 起動時の初期化処理。routes読み込み、upload準備などを行う。 |
| `application/configs/application.ini` | アプリ設定。Controller、Layout、View、DB接続などを定義する。 |
| `application/configs/routes.php` | URLとController/Actionの対応を定義する。 |
| `application/controllers` | URLごとの処理を受けるControllerを置く。 |
| `application/forms` | Zend_Formで作られたフォーム定義を置く。 |
| `application/models` | チケットのデータモデルとDB操作用Mapperを置く。 |
| `application/models/DbTable` | DBテーブルに対応するZend_Db_Tableクラスを置く。 |
| `application/views/scripts` | 画面テンプレートの `.phtml` を置く。 |
| `application/views/helpers` | Viewから呼び出す補助クラスを置く。 |
| `application/layouts/scripts/layout.phtml` | 共通レイアウト。ナビゲーションや共通HTMLを定義する。 |
| `application/helpers` | Controllerから呼び出すAction Helperを置く。 |
| `application/uploads` | アップロード画像の保存先。 |
| `tests` | PHPUnit用のテスト設定とテストクラス。 |
| `other_example` | 参考画像や別サンプルのアーカイブ。 |

## ZF1として重要なファイル

### `public/index.php`

全リクエストの入口。

`Zend_Application` を作成し、`application.ini` と `Bootstrap.php` を読み込んでアプリを起動する。

### `application/Bootstrap.php`

起動時の初期化処理。

このアプリでは、doctype設定、routes.php読み込み、uploadディレクトリの準備を行う。

### `application/configs/routes.php`

URLとController/Actionの対応を定義する。

トップページ `/` は `TicketController#indexAction` に割り当てられている。

## MVCごとのファイル配置

### Controller

```text
application/controllers
├─ TicketController.php
├─ IndexController.php
└─ ErrorController.php
```

中心は `TicketController.php`。

チケット一覧、新規登録、編集、削除、CSV出力を担当する。

### Model

```text
application/models
├─ Ticket.php
├─ TicketMapper.php
└─ DbTable
   └─ TicketDbTable.php
```

`Ticket.php` はチケット1件分のデータ入れ物。

`TicketMapper.php` はControllerとDBの橋渡し。

`TicketDbTable.php` はDBテーブル `tickets` に対応する。

### View

```text
application/views/scripts
├─ controls.phtml
├─ error
│  └─ error.phtml
├─ index
│  └─ index.phtml
└─ ticket
   ├─ delete.phtml
   ├─ edit.phtml
   ├─ index.phtml
   └─ save.phtml
```

チケット機能で主に使うのは `ticket` 配下。

`controls.phtml` は一覧画面のページング部品。
