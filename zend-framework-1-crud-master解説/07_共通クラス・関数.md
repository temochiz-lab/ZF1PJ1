# 共通クラス・関数

対象アプリ: `zend-framework-1-crud-master`

このページでは、アプリ内で共通的に使われている独自クラス、独自ヘルパー、共通的な処理単位を階層化して整理する。

Zend Framework 1の標準クラスや標準View Helperそのものの説明はここでは扱わない。個別ファイルの詳しい解説はリンク先に寄せる。

## 全体像

```text
Controller
├─ Form層
│  └─ Application_Form_TopicBootstrapForm
├─ Model層
│  ├─ Application_Model_Ticket
│  ├─ Application_Model_TicketMapper
│  └─ Application_Model_DbTable_TicketDbTable
├─ Action Helper
│  ├─ Csv
│  └─ Multiples
└─ View / Layout
   ├─ displayDate()
   ├─ actionErrors()
   ├─ displayAddress()
   ├─ loggedInUser()
   └─ menuList()
```

このアプリで実際に重要なのは、次の流れ。

```text
TicketController
↓
TopicBootstrapForm
↓
Ticket
↓
TicketMapper
↓
TicketDbTable
↓
tickets テーブル
```

## 1. Model層

チケットデータをPHPオブジェクトとして持ち、DBテーブルとの出し入れを担当する層。

### 1.1 `Application_Model_Ticket`

| 項目 | 内容 |
| --- | --- |
| 種類 | データ保持用モデル |
| 定義 | [Ticket.php 解説](各ファイル解説/application/models/Ticket.php_解説.md) |
| 主な利用元 | [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md)、[TicketMapper.php 解説](各ファイル解説/application/models/TicketMapper.php_解説.md) |
| 対応データ | `tickets` テーブル1行分 |

`Ticket` はチケット1件分の値を持つ入れ物。DB行やフォーム入力値を、そのまま配列で扱わず、`getTitle()` や `setNotes()` のようなメソッド経由で扱えるようにする。

主なプロパティ:

| プロパティ | DBカラム | 役割 |
| --- | --- | --- |
| `id` | `id` | チケットID |
| `title` | `title` | タイトル |
| `notes` | `notes` | 本文 |
| `created_at` | `created_at` | 作成日時 |
| `updated_at` | `updated_at` | 更新日時 |
| `files` | `files` | アップロードファイル名 |
| `priority` | `priority` | 優先度 |

読むポイント:

- `setOptions()` に配列を渡すと、キー名に対応したsetterを呼ぶ。
- `TicketController#saveAction()` では、フォーム値から `Application_Model_Ticket` を作る。
- `TicketMapper#fetchAllTopics()` では、DB行から `Application_Model_Ticket` を作る。

### 1.2 `Application_Model_TicketMapper`

| 項目 | 内容 |
| --- | --- |
| 種類 | DB操作の仲介役 |
| 定義 | [TicketMapper.php 解説](各ファイル解説/application/models/TicketMapper.php_解説.md) |
| 主な利用元 | [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md) |
| 内部で使うクラス | `Application_Model_Ticket`、`Application_Model_DbTable_TicketDbTable` |

`TicketMapper` はControllerとDBアクセスを分離するための中心クラス。ControllerはSQLやDbTableを直接扱わず、このMapperを通してチケットを取得・保存・削除する。

主なメソッド:

| メソッド | 入力 | 戻り値/結果 | 主な用途 |
| --- | --- | --- | --- |
| `setDbTable($dbTable)` | DbTableクラス名またはインスタンス | 自分自身 | 利用するDbTableを差し替える |
| `getDbTable()` | なし | `TicketDbTable` | DB操作用オブジェクトを取得する |
| `saveTopic($ticket)` | `Application_Model_Ticket` | insert/update結果 | 新規登録または更新 |
| `findTopic($id, $ticket)` | ID、空のTicketモデル | 値を詰めたTicketモデル | 編集画面用の1件取得 |
| `fetchAllTopics()` | なし | Ticketモデル配列 | 一覧表示用 |
| `fetchAllCvs()` | なし | 連想配列 | CSV出力用 |
| `deleteTopic($id)` | ID | delete結果 | 削除 |

処理上の分岐:

```text
saveTopic()
├─ idなし → insert
└─ idあり → update
```

注意点:

- `deleteTopic($id)` は `"id = $id"` の文字列条件で削除している。実運用ならプレースホルダやquote処理を使う方が安全。
- `fetchAllTopics()` は画面用に `Ticket` オブジェクトへ変換する。
- `fetchAllCvs()` はCSV用に配列のまま返す。ここが一覧表示用との違い。

### 1.3 `Application_Model_DbTable_TicketDbTable`

| 項目 | 内容 |
| --- | --- |
| 種類 | DBテーブル対応クラス |
| 定義 | [TicketDbTable.php 解説](各ファイル解説/application/models/DbTable/TicketDbTable.php_解説.md) |
| 主な利用元 | [TicketMapper.php 解説](各ファイル解説/application/models/TicketMapper.php_解説.md) |
| 対応テーブル | `tickets` |

このクラス自体は薄く、実質的には次の指定が中心。

```php
protected $_name = 'tickets';
```

`TicketMapper` から呼ばれることで、`tickets` テーブルに対する `insert()`、`update()`、`find()`、`fetchAll()`、`delete()` が使われる。

## 2. Form層

登録・編集画面の入力項目、バリデーション、ファイルアップロード設定をまとめる層。

### 2.1 `Application_Form_TopicBootstrapForm`

| 項目 | 内容 |
| --- | --- |
| 種類 | チケット登録/編集フォーム |
| 定義 | [TopicBootstrapForm.php 解説](各ファイル解説/application/forms/TopicBootstrapForm.php_解説.md) |
| 主な利用元 | [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md)、[save.phtml 解説](各ファイル解説/application/views/scripts/ticket/save.phtml_解説.md)、[edit.phtml 解説](各ファイル解説/application/views/scripts/ticket/edit.phtml_解説.md) |

`TopicBootstrapForm` は、画面のHTMLフォームをPHP側で組み立てるクラス。登録画面と編集画面の両方で使われる。

フォーム項目:

| 項目 | 型/用途 | 補足 |
| --- | --- | --- |
| `id` | hidden | 編集時のID保持 |
| `title` | text | 必須。チケットタイトル |
| `notes` | textarea | 必須。チケット本文 |
| `priority` | select | Low / Normal / High / Emergency |
| `files` | file | jpg/png/gifを許可 |
| `csrf` | hash | CSRF対策 |

画面との関係:

```text
新規登録
TicketController#saveAction()
↓
TopicBootstrapForm
↓
ticket/save.phtml

編集
TicketController#editAction()
↓
TopicBootstrapForm に既存値を populate()
↓
ticket/edit.phtml
↓
POST先は ticket/create
```

注意点:

- `files` が必須になっているため、編集時もファイル入力が絡む。
- `UPLOAD_PATH` 定数に依存する。定数は [Bootstrap.php 解説](各ファイル解説/application/Bootstrap.php_解説.md) 側で触れている。
- 見た目はBootstrap向けのclassとdecoratorで調整されている。

### 2.2 `Application_Form_TopicForm`

| 項目 | 内容 |
| --- | --- |
| 種類 | 旧フォームまたは未使用フォーム |
| 定義 | [----TopicForm.php 解説](各ファイル解説/application/forms/----TopicForm.php_解説.md) |

## 3. Action Helper

Controllerから `$this->_helper->...` の形で呼び出す補助処理。

### 3.1 `Csv`

| 項目 | 内容 |
| --- | --- |
| 種類 | CSVダウンロード用Action Helper |
| 定義 | [Csv.php 解説](各ファイル解説/application/helpers/Csv.php_解説.md) |
| 主な利用元 | [TicketController.php 解説](各ファイル解説/application/controllers/TicketController.php_解説.md) |
| 呼び出し箇所 | `TicketController#cvsexportAction()` |

呼び出し:

```php
$this->_helper->Csv($blogTopic->fetchAllCvs(), "tickets");
```

処理の流れ:

```text
TicketController#cvsexportAction()
↓
TicketMapper#fetchAllCvs()
↓
Csv Helper
↓
CSV用HTTPヘッダ
↓
tickets-export.csv を直接出力
```

役割:

- Viewテンプレートを使わず、レスポンスをCSVとして直接作る。
- 配列データの先頭にカラム名を付ける。
- ダウンロードファイル名を組み立てる。
- ExcelでCSVを開く場合の誤認識対策も含む。

| 項目 | 内容 |
| --- | --- |

## 4. View Helper

Viewテンプレートから `$this->helperName()` の形で呼び出す表示補助。

### 4.1 実際に主要画面で使われるHelper

#### `displayDate()`

| 項目 | 内容 |
| --- | --- |
| 種類 | 日付表示用View Helper |
| 定義 | [DisplayDate.php 解説](各ファイル解説/application/views/helpers/DisplayDate.php_解説.md) |
| 主な利用元 | [ticket/index.phtml 解説](各ファイル解説/application/views/scripts/ticket/index.phtml_解説.md) |

## 5. Layout / View部品との関係

共通クラスではないが、画面全体で繰り返し効く共通部品。

| 部品 | 解説 | 役割 |
| --- | --- | --- |
| `layout.phtml` | [layout.phtml 解説](各ファイル解説/application/layouts/scripts/layout.phtml_解説.md) | 全画面共通レイアウト、ナビゲーション、FlashMessage表示 |
| `controls.phtml` | [controls.phtml 解説](各ファイル解説/application/views/scripts/controls.phtml_解説.md) | 一覧ページングの共通View |

`layout.phtml` は全画面に効くため、画面表示の共通部品として重要。`controls.phtml` は `ticket/index.phtml` からページング表示に使われる。

## 8. このページでの分類

| 分類 | 本命度 | 読む優先度 |
| --- | --- | --- |
| `TicketController` + `TopicBootstrapForm` + `TicketMapper` | 高 | 最初に読む |
| `Ticket` + `TicketDbTable` | 高 | データの流れを見る時に読む |
| `Csv` | 中 | CSV出力を見る時に読む |
| `displayDate()` + `controls.phtml` | 中 | 一覧表示を見る時に読む |
| 未使用・流用元系Helper | 低 | コード整理や不要物確認の時に読む |
