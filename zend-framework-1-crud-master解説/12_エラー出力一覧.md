# エラー出力一覧

対象アプリ: `zend-framework-1-crud-master`

## 概要

このアプリのエラー出力は、主に次の4種類。

| 種類 | 主な場所 | 内容 |
| --- | --- | --- |
| フレームワークエラー | [`ErrorController.php`](各ファイル解説/application/controllers/ErrorController.php_解説.md) | 404、500など |
| 画面エラー | [`error/error.phtml`](各ファイル解説/application/views/scripts/error/error.phtml_解説.md) | 例外情報、リクエスト情報 |
| フォームエラー | [`TopicBootstrapForm.php`](各ファイル解説/application/forms/TopicBootstrapForm.php_解説.md) | 必須入力、ファイル拡張子、CSRFなど |
| 処理結果メッセージ | [`TicketController.php`](各ファイル解説/application/controllers/TicketController.php_解説.md) + [`layout.phtml`](各ファイル解説/application/layouts/scripts/layout.phtml_解説.md) | 登録・更新・削除の成功/失敗 |

## ErrorControllerのエラー分類

| エラー種別 | HTTPステータス | 表示メッセージ | ログ優先度 |
| --- | --- | --- | --- |
| ルートなし | 404 | `Page not found` | `Zend_Log::NOTICE` |
| Controllerなし | 404 | `Page not found` | `Zend_Log::NOTICE` |
| Actionなし | 404 | `Page not found` | `Zend_Log::NOTICE` |
| その他の例外 | 500 | `Application error` | `Zend_Log::CRIT` |
| `error_handler` なし | 不明 | `You have reached the error page` | なし |

## エラー画面

ファイル:

- [`application/views/scripts/error/error.phtml`](各ファイル解説/application/views/scripts/error/error.phtml_解説.md)

表示内容:

- `An error occurred`
- `$this->message`
- 例外情報
- スタックトレース
- リクエストパラメータ

ただし、例外情報は `displayExceptions` が有効な場合に表示される。

## displayExceptions設定

[`application/configs/application.ini`](各ファイル解説/application/configs/application.ini_解説.md) では、例外表示が有効になっている。

```ini
phpSettings.display_errors = 1
resources.frontController.params.displayExceptions = 1
```

注意:

- 学習・解析用途では見やすい。
- 実運用では内部情報が画面に出るため危険。
- 本番では `display_errors = 0`、`displayExceptions = 0` を検討する。

## フォームエラー

ファイル:

- [`application/forms/TopicBootstrapForm.php`](各ファイル解説/application/forms/TopicBootstrapForm.php_解説.md)

主なエラー:

| 項目 | 条件 | エラー内容 |
| --- | --- | --- |
| `title` | 必須 | 未入力ならエラー |
| `notes` | 必須 | 未入力ならエラー |
| `priority` | `NotEmpty` | 未選択ならエラー |
| `files` | 必須 | 未指定ならエラー |
| `files` | 拡張子 | `jpg,png,gif` 以外ならエラー |
| `csrf` | hash | CSRF検証エラー |

## 処理結果メッセージ

`TicketController` では、登録・更新・削除後に `FlashMessenger` へメッセージを入れる。

[`layout.phtml`](各ファイル解説/application/layouts/scripts/layout.phtml_解説.md) がそのメッセージをBootstrap alertとして表示する。

| 処理 | 成功時 | 失敗時 |
| --- | --- | --- |
| 登録 | `Ticket has been successfully posted.` | `Some error occurred while posting tickets, Please try after some time.` |
| 更新 | `Ticket has been successfully updated.` | `Some error occurred while updating ticxket, Please try after some time.` |
| 削除 | `Ticket has been successfully deleted.` | `Some error occurred while deleting ticket, Please try after some time.` |

## 例外が起きる可能性がある箇所

| ファイル | 箇所 | 内容 |
| --- | --- | --- |
| [`Bootstrap.php`](各ファイル解説/application/Bootstrap.php_解説.md) | `_initUploadDirAndConstant()` | `application/uploads` を作れない場合に例外 |
| [`Ticket.php`](各ファイル解説/application/models/Ticket.php_解説.md) | `__set()` / `__get()` | 未定義プロパティアクセスで例外 |
| [`TicketMapper.php`](各ファイル解説/application/models/TicketMapper.php_解説.md) | `setDbTable()` | `Zend_Db_Table_Abstract` 以外なら例外 |
| [`TicketController.php`](各ファイル解説/application/controllers/TicketController.php_解説.md) | `saveAction()` | ファイルアップロード例外をcatch |

## 注意点

- `TicketController#saveAction()` のアップロード例外はcatchしているが、ログや画面通知に使っていない。
- 削除・登録・更新失敗時の原因は画面メッセージだけでは分からない。
- `ErrorController` のログ処理はあるが、ログリソース設定がない可能性がある。
